The goal of VeriSIM is to help you understand software design diagrams

WHY IS IT IMPORTANT? (Some change in text)

When you graduate and enter the software industry, you will be spending your first several months resolving bugs and writing additional features based on new requirements


These activities will require you to explicitly understand the software design

DESIGN TRACING(Some change in text)

In VeriSIM, you will be introduced to a technique called design tracing.

Design tracing will help you trace different scenarios in a given software design.

You will trace scenarios by constructing a state diagram

Tracing different scenarios in a design can help you understand the design better and also uncover defects in the design

LEARNING OBJECTIVES
(Some change in text) By the end of this session, you should be able to
  1. Understand the purpose of the design tracing technique to trace a given scenario
  2. Apply the design tracing technique to trace a given scenario by constructing a state diagram
  3. Visualize how data and events can be used to trace a given scenario

Don't worry if all this seems new to you!
