WELCOME TO THE VeriSIM LEARNING ENVIRONMENT!


The goal of VeriSIM is to help you comprehend object-oriented design diagrams (UML diagrams)

WHY IS IT IMPORTANT?

When students graduate and enter the software industry, they spend their first several months resolving bugs and writing additional features based on new requirements


These activities require students to explicitly understand software designs

DESIGN TRACING

In VeriSIM, you will be introduced to a technique called design tracing.

Design tracing will help you model different scenarios in a given software design.

You will model scenarios by constructing a state diagram

Tracing different scenarios in a design can help you understand the design better and also help you uncover defects in the design

LEARNING OBJECTIVES
By the end of this session, you should be able to
  1. Understand the purpose of the design tracing technique to model a given scenario
  2. Apply the design tracing technique to model a given scenario by constructing a state diagram
  3. Visualize how data and events model a given scenario

Don't worry if all this seems new to you!
